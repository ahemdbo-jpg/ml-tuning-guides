üîç

GridSearchCV

üê¢ Slow

The exhaustive searcher. Tests every single combination in your parameter grid. Guaranteed to find the absolute best parameters, but can be very time-consuming.

Search Strategy: Test ALL combinations

‚úì Pros

  • Guaranteed best result
  • Systematic & reproducible
  • Built-in cross-validation
  • Easy to understand

‚úó Cons

  • Very slow for large grids
  • Exponential time complexity
  • Wastes time on bad areas
  • Limited by discrete values

üí° Best Used When:

  • Small parameter space (< 100 combinations)
  • You need the guaranteed best result
  • Computational resources aren't limited
  • Parameters are few (2-3 hyperparameters)
üé≤

RandomizedSearchCV

üöÄ Fast

The smart sampler. Randomly samples from parameter distributions. Often finds near-optimal solutions much faster than grid search.

Search Strategy: Sample RANDOM combinations

‚úì Pros

  • Much faster than Grid
  • Explores continuous distributions
  • Good for large spaces
  • Often finds near-optimal

‚úó Cons

  • Not guaranteed best
  • Results can vary
  • May miss optimal region
  • Needs more iterations

üí° Best Used When:

  • Large parameter space (> 100 combinations)
  • Limited time or compute budget
  • Many hyperparameters (5+)
  • "Good enough" is acceptable
‚ö°

HalvingGridSearchCV

‚ö° Medium

The tournament organizer. Starts with all combinations using minimal resources, then successively eliminates poor performers while increasing resources for survivors.

Search Strategy: Successive HALVING elimination

Round 1: Test all 50 combinations (10% data)
‚Üì Eliminate worst 2/3
Round 2: Test best 16 combinations (30% data)
‚Üì Eliminate worst 2/3
Round 3: Test best 5 combinations (100% data)
‚Üì
üèÜ Winner!

‚úì Pros

  • Faster than GridSearch
  • Eliminates bad options early
  • Still exhaustive search
  • Efficient resource use

‚úó Cons

  • Complex to configure
  • May eliminate good options early
  • Requires scikit-learn 0.24+
  • Not all models supported

üí° Best Used When:

  • Medium parameter space (50-500 combinations)
  • Large dataset available
  • Want systematic search faster
  • Model training is expensive
üöÄ

HalvingRandomSearchCV

‚ö°‚ö° Very Fast

The speed champion. Combines random sampling with successive halving. The fastest method for very large parameter spaces.

Search Strategy: RANDOM samples + HALVING

üé≤ Randomly sample 100 combinations
‚Üì
Round 1: Test 100 (10% data)
‚Üì Eliminate worst 2/3
Round 2: Test best 33 (30% data)
‚Üì Eliminate worst 2/3
Round 3: Test best 11 (100% data)
‚Üì
üèÜ Winner!

‚úì Pros

  • Fastest of all methods
  • Great for huge spaces
  • Combines best of both
  • Efficient early stopping

‚úó Cons

  • Most complex to use
  • Results vary more
  • Requires careful tuning
  • Newest/least tested

üí° Best Used When:

  • Very large parameter space (1000+ combinations)
  • Severe time/compute constraints
  • Many hyperparameters (10+)
  • Initial exploration phase

4+ parameters (> 100 combos) ‚Üí Continue below ‚Üì

from sklearn.model_selection import GridSearchCV
from sklearn.ensemble import RandomForestClassifier

# Define parameter grid
param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth': [5, 10, 15, 20],
    'min_samples_split': [2, 5, 10]
}

# Create GridSearchCV
grid_search = GridSearchCV(
    estimator=RandomForestClassifier(),
    param_grid=param_grid,
    cv=5,                    # 5-fold cross-validation
    scoring='accuracy',
    n_jobs=-1,              # Use all CPU cores
    verbose=2
)

# Fit and find best parameters
grid_search.fit(X_train, y_train)

# Results
print("Best Parameters:", grid_search.best_params_)
print("Best Score:", grid_search.best_score_)

from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint, uniform

# Define parameter distributions
param_distributions = {
    'n_estimators': randint(50, 500),           # Random integers
    'max_depth': randint(5, 50),
    'min_samples_split': randint(2, 20),
    'max_features': uniform(0.1, 0.9)          # Random floats
}

# Create RandomizedSearchCV
random_search = RandomizedSearchCV(
    estimator=RandomForestClassifier(),
    param_distributions=param_distributions,
    n_iter=100,             # Number of random combinations
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42,
    verbose=2
)

# Fit and find best parameters
random_search.fit(X_train, y_train)

print("Best Parameters:", random_search.best_params_)
print("Best Score:", random_search.best_score_)
from sklearn.experimental import enable_halving_search_cv
from sklearn.model_selection import HalvingGridSearchCV

# Define parameter grid
param_grid = {
    'n_estimators': [50, 100, 200],
    'max_depth': [5, 10, 15, 20],
    'min_samples_split': [2, 5, 10]
}

# Create HalvingGridSearchCV
halving_search = HalvingGridSearchCV(
    estimator=RandomForestClassifier(),
    param_grid=param_grid,
    factor=3,               # Reduce candidates by 1/3 each iteration
    resource='n_samples',   # Increase training data each round
    max_resources='auto',
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    verbose=2
)

# Fit and find best parameters
halving_search.fit(X_train, y_train)

print("Best Parameters:", halving_search.best_params_)
print("Best Score:", halving_search.best_score_)
from sklearn.experimental import enable_halving_search_cv
from sklearn.model_selection import HalvingRandomSearchCV
from scipy.stats import randint, uniform

# Define parameter distributions
param_distributions = {
    'n_estimators': randint(50, 500),
    'max_depth': randint(5, 50),
    'min_samples_split': randint(2, 20),
    'max_features': uniform(0.1, 0.9)
}

# Create HalvingRandomSearchCV
halving_random = HalvingRandomSearchCV(
    estimator=RandomForestClassifier(),
    param_distributions=param_distributions,
    factor=3,
    resource='n_samples',
    n_candidates='exhaust',  # Use all available candidates
    cv=5,
    scoring='accuracy',
    n_jobs=-1,
    random_state=42,
    verbose=2
)

# Fit and find best parameters
halving_random.fit(X_train, y_train)

print("Best Parameters:", halving_random.best_params_)
print("Best Score:", halving_random.best_score_)
